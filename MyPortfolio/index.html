Project 0: The <Joe> Button

In this project, I will design, implement and document via this web portfolio a software application where central interaction is with a button, which represents some aspects of my client's personality or character.
As a group of five, we initially decided to interview our classmates separately, and then collect our ideas and use a celebrity as our client. After finishing ten sketches, comparing and discussing, we decided to create a button that represents our classmate Robert.

Project 1: Smartphone Interaction Workaround

Sketch, design, implement and document via a portfolio an animated touch-based and sensor based interface for a mobile phone that is an alternative to ‘slide to unlock’.
As a group of five, we firstly sketched ten original sketches on our own and then discussed together and picked roughly 5 from all of our ideas. Next, we present the refinement sketches to the class and received some critiques. Finally, we made the unlock app using Android Studio.

Refined Sketches

Everyone in our group has come up with very interesting ideas. We picked the most interesting and feasible five ones to work on. They are "Defusing Bomb" from Matt, "Gas Pedal" from Jason, "Where's Waldo" form Karn, "Eyes Opening" from Jason, "Water Draining" and "Fruit Ninja" from me. Sensor-Based

The sensor-based unlock interface uses the light sensor to control the eyes opening and closing animation on the screen. When using fingers to cover the light sensor, the lux value decrements to a certain range, and the eyes will close. The pupils will move following the movement of the phone. After the eyes' movement and blink, the screen will then unlock. As an additional feature, you can change the eye styles by shaking the phone.
This interface can avoid accidental unlock since the user need to follow a certain eye movement pattern (e.g., in our demo, left-most to right-most and close) to unlock.

Touch-Based

The touch-based unlock interface is a variation of the "Where's Waldo" game. Different Waldo pictures are placed on the photos from the user's photo gallery. The user need to click all the Waldo'es on the screen to successfully unlock the phone.
This interface can avoid accidental unlock since the user need to click a series positions on the screen (e.g., where the Waldo is) to unlock.
